## 1. 사전 준비

### 필요한 프로그램
| 프로그램 | 버전 | 다운로드 |
|---|---|---|
| Git | 최신 | https://git-scm.com |
| Node.js | 18 이상 | https://nodejs.org |
| Python | 3.10 이상 | https://python.org |

### OpenAI API 키
- https://platform.openai.com 에서 발급
- 프롬프트 생성(GPT)과 이미지 생성(DALL-E / gpt-image) 모두 동일 키 사용
- **주의**: ChatGPT Plus 구독과 API는 별개 — API 크레딧 별도 충전 필요
- 크레딧 충전: https://platform.openai.com/settings/organization/billing

## 2. 설치 — Mac

### 터미널에서 실행
```bash
# 원하는 폴더로 이동 후
git clone https://github.com/Dogoon/handyandy.git
cd handyandy
npm install
```

### 실행
`start.command` 파일을 더블클릭

> 처음 실행 시 보안 경고가 뜰 수 있습니다.
> **시스템 설정 → 개인 정보 보호 및 보안 → 확인 없이 열기** 허용

### 종료
`start.command` 터미널 창을 닫으면 웹서버와 에이전트가 함께 종료됩니다.

## 3. 설치 — Windows

### 설치 시 주의사항
- Python 설치 시 **"Add Python to PATH"** 체크 필수
- Git 설치 시 기본 옵션으로 설치

### 명령 프롬프트에서 실행
```
git clone https://github.com/Dogoon/handyandy.git
cd handyandy
npm install
```

### 실행
`start.bat` 파일을 더블클릭

> Python 명령어 오류 시 `start.bat`을 메모장으로 열어
> `python`을 `py`로 변경 후 저장

### 종료
시작 창 + 웹서버 창 + 에이전트 창 각각 닫기

## 4. 최초 설정

앱 실행 후 브라우저에서 `http://localhost:3000` 자동으로 열립니다.

### API 키 입력
1. 왼쪽 사이드바 하단 **설정** 메뉴 이동
2. **API 키 관리** 섹션에서 ChatGPT API 키 입력
3. 저장

### AI 모델 설정
- **ChatGPT 모델**: 프롬프트 생성에 사용. 기본값 `gpt-4o-mini`
- **이미지 생성 모델**: 기본값 `gpt-image-1`
- **모델 목록 갱신** 버튼으로 최신 모델 목록 가져오기 가능

## 5. 프로젝트 관리

### 새 프로젝트 추가
1. 설정 → 프로젝트 관리 → **새 프로젝트 추가**
2. 프로젝트 코드 입력 (예: `BECK`)
3. 프로젝트 이름 입력
4. EP / SC / CUT 구조 체크 (없는 항목은 체크 해제)
5. **상위 폴더 선택** → 프로젝트를 만들 상위 폴더 선택
6. 확인 → 폴더 구조 자동 생성

> 생성되는 폴더 구조:
> ```
> [선택한폴더]/[프로젝트코드]/
> ├── prompts/
> ├── images/
> ├── presets/
> └── _assets/
>     ├── prompts/ (characters, props, backgrounds, design)
>     └── images/
> ```

### 기존 프로젝트 연결
경로가 없는 프로젝트는 설정에서 편집(✏️) 버튼 → 상위 폴더 선택으로 연결

### 프로젝트 전환
사이드바 상단 프로젝트 이름 클릭 → 드롭다운에서 선택
마지막 선택한 프로젝트는 다음 실행 시 자동 복원됩니다.

## 6. 프롬프트 생성

### 작업 유형 선택
| 유형 | 설명 |
|---|---|
| 디자인 이미지 | 컨셉 아트, 배경 디자인 |
| 캐릭터 시트 | 캐릭터 디자인 |
| 소품 | 소품/오브젝트 |
| 배경 | 배경 이미지 |
| 컷 이미지 | 샷/컷 기반 이미지 |
| 영상 | 영상 프롬프트 |

### 정보 입력
- **EP / SC / CUT**: 프로젝트 구조에 따라 표시 (비활성 항목은 숨김)
- **어셋명**: 파일명에 포함되는 이름
- **프롬프트 항목**: Scene, Composition, Light, Camera, Art Style, Ratio, Negative

### 생성 및 저장
1. **프롬프트 생성** → GPT가 영문/한국어 프롬프트 생성
2. 생성된 프롬프트 확인 (영문/한국어 옆 복사 아이콘으로 복사 가능)
3. **프롬프트 저장** → JSON 파일로 저장
   - 같은 이름 파일 존재 시: **취소 / 버전업 / 덮어쓰기** 선택
   - 버전업: `V001.json` → `V001.0001.json` → `V001.0002.json`
4. **이미지 생성** → 저장된 프롬프트로 이미지 생성 (프롬프트 저장 후 활성화)

### 파일 확인
- **이미지 열기**: 생성된 이미지를 OS 기본 뷰어로 열기
- **폴더 열기**: 이미지 저장 폴더를 Finder/탐색기로 열기

## 7. 프리셋 저장 및 불러오기

작업 유형, 어셋 정보, 프롬프트 항목 설정값을 저장해두고 재사용할 수 있습니다.

### 저장
1. 원하는 설정값 입력 후 **설정 저장** 버튼
2. 프로젝트 루트의 `presets/` 폴더에 `TPL_파일명.json`으로 저장

### 불러오기
1. **설정 불러오기** 버튼 → 파일 선택 창 열림
2. 저장해둔 `TPL_*.json` 파일 선택
3. 선택 즉시 모든 설정값 적용

---

## 8. 프롬프트 검색

1. 사이드바 **프롬프트 검색** 메뉴
2. 검색 범위 선택: 현재 프로젝트 / 전체 / 특정 프로젝트
3. 검색어 입력 후 Enter 또는 검색 버튼
   - 파일명, 프롬프트 내용, 어셋명으로 검색 가능
4. 결과 카드에서 썸네일 확인, **경로 열기**로 파일 위치 이동

## 문제 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 연결 끊김 표시 | 에이전트 미실행 | start 파일로 재실행 |
| 프롬프트 생성 실패 | API 키 오류 또는 크레딧 부족 | 설정에서 API 키 확인, 크레딧 충전 |
| 이미지 생성 실패 "does not exist" | 계정 미지원 모델 | 설정에서 이미지 모델을 `gpt-image-1`으로 변경 |
| 폴더 열기 안 됨 | 폴더 미생성 | 자동 생성됨, 재시도 |
| Windows Python 오류 | PATH 미설정 | Python 재설치 시 "Add to PATH" 체크 |
